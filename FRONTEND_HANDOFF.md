# Frontend Handoff — HomeGuide AI

## Your job

Build a **chat UI** for a real estate assistant. Users type what they want; the backend searches homes and streams back text + property cards.

**You design the UI.** The `frontend/` folder is only a small reference — rebuild it your way.

---

## What to build

- Chat screen (user messages + assistant messages)
- Text input + send
- **New chat** button
- Property cards when homes are returned (price, beds/baths, address, city)
- Loading state while waiting

**Don't build:** filter forms, listing browse page, map, login.

---

## Connect to the API

```
POST {API_URL}/chat/stream
```

```json
{
  "session_id": "uuid-in-localStorage",
  "message": "Modern condo in Austin under 600k"
}
```

Set in your `.env`:

```env
VITE_API_URL=http://127.0.0.1:8000
```

Response is a **stream** (SSE). Each line looks like:

```
data: {"type":"status","content":"Searching..."}
data: {"type":"text","content":"I found "}
data: {"type":"properties","properties":[...]}
data: {"type":"done"}
```

| Event | Do this |
|-------|---------|
| `status` | Show loading text |
| `text` | Add to assistant message |
| `properties` | Show property cards |
| `done` | Stop loading |
| `error` | Show error |

**Session:** create a UUID, save in `localStorage` as `session_id`, send it every message. New chat = new UUID.

**Property card fields:** `price`, `beds`, `baths`, `address`, `city`, `state`, `neighborhood` (zip), `property_type`, `description`. No photos.

**Reference code:** `frontend/src/main.js` has a working example.

## Run the backend (on your laptop)

You need the API running while you work. Not on the same WiFi as someone else? **Run it yourself** — don't connect to their laptop.

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Add keys to `.env` (ask backend dev or get your own):
- `GOOGLE_API_KEY` — https://aistudio.google.com/apikey
- `RENTCAST_API_KEY` — https://app.rentcast.io

```bash
uvicorn main:app --reload
python main.py Austin TX 100   # load listings
```

- API: http://127.0.0.1:8000
- Docs: http://127.0.0.1:8000/docs
